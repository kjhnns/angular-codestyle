Conventions for Frontend Development:

Controller
"controllers/NameCtrl.js" - CamelCase, should end on "Ctrl"
Services
"services/NameService.js" - CamelCase, should end on "Service"
Directives
"directives/NameDirective.js" - CamelCase, should end on "Directive"
Filters
"filters/NameFilter.js" - CamelCase, should end on "Filter"
States
Lowercase, should get nested by ".", e.g. "parent.child", "frontend.main"
logic = controller
Put the logic that you need for your code inside a Controller. Controller are meant to keep logic from your views away (except for repetitions).
view name = state
Especially when it comes to nested states, e.g. let dummy be the parent state and index be a child states: dummy.index.html instead of index.html
no ng-init
Don't use the ng-init tag in views. There is actually no need for the initialization since you can use your controllers/services to initialize data etc. Besides that it makes your code hard to read.
controller as
if you ever feel like you need to use ng-controller, please make sure you are binding the controller to a variable, e.g. ng-controller="SettingsController as settings". Controllers are meant to be injected by the $stateProvider.
no inline style
Don't use the "style" tag/attribute. Always create a css class. They keep the code clean and help people to reuse your code.
